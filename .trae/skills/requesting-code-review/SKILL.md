# Requesting Code Review

Run a structured review against the plan. Identify issues by severity; block on critical problems, and record actions to address findings before proceeding.

## Triggers
- Between tasks or prior to merge/PR.

## Guidance
- Check spec compliance first; then code quality and tests.
- Assign severity: critical blocks, major needs fixes, minor optional.
- Document findings and remediation steps.

## Output
- Review report and actionable fixes.
